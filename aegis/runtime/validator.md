# Validator (Consensus Engine)

## The Final Review
The Validator acts as the ultimate gatekeeper before the AI responds to the user.

1. **Virtual Team Consensus**: 
   - If the task is purely cosmetic, skip.
   - If the task involves architectural shifts, security, or core performance, the Validator MUST invoke `architecture_agent.md`, `security_agent.md`, and `performance_agent.md` to debate the simulated solution.
2. **Approval**: Only if consensus is reached, the execution phase begins.
3. **Execution**: The AI writes the code, updates the files, or provides the final answer to the user.
4. **Handoff**: Pass the execution result to `learning_engine.md` for reflection.
